Follow below step-by-step to create Data Access Layer(DAL) methods using Boto3(resource) dynamodb in separate file:
1) import boto3 python library
2) Data transfer objects(dto) are present in dto folder, data access layer (dal) will be present in dal folder, unit test script will be present in test folder. All dto, model, dal and test folders are present in data folder. This point is only information not to perform any action.
3) Table name coming from environment variable {{ddb_table_name_env_variable}} using python os library 
4) region coming from environment variable AWS_REGION using python os library 
{{time_to_live_replacement}}
5) Create below DAL methods by following below steps. Methods should use tenacity's retry python library to retry with exponential backoff. Use number of retries and backoff time stored in variables. All the methods should have exception handling and @retry decorator.
6) First method is to put an item in {{ddb_table_name}} DynamoDB table with name {{ddb_table_name_lower}}_put_item. Input to the function is variable of {{ddb_table_name}}Dto data transfer objects{{created_at_replacement}}. version attribute has to set to 1 and time_to_live attribute has to be set if get_ttl() method is present.
7) Second method is to update an item in {{ddb_table_name}} DynamoDB table with name {{ddb_table_name_lower}}_update_item. Input to the function is variable of {{ddb_table_name}}Dto data transfer objects{{updated_at_replacement}}. Use optimistic locking with version attribute 
8) Third method is to get an item from {{ddb_table_name}} DynamoDB table with name {{ddb_table_name_lower}}_get_item. Input to the function is partition/hash key (pk){{ddb_table_sk}}.
9) Fourth method is to delete an item in {{ddb_table_name}} DynamoDB table with name {{ddb_table_name_lower}}_delete_item. Input to the function is partition/hash key (pk){{ddb_table_sk}}. Use optimistic locking with version attribute. 
10) Fifth method is to create_or_update an item in {{ddb_table_name}} DynamoDB table with name {{ddb_table_name_lower}}_create_or_update_item. Input to the function is variable of {{ddb_table_name}}Dto data transfer objects. This method will get the item first, if item does not exist it will create item else update the attributes of the get item with dto values{{created_at_replacement}} and {{updated_at_replacement}}. Use the update method created earlier which already uses optimistic locking.
11) Sixth method is to query {{ddb_table_name}} DynamoDB table with name {{ddb_table_name_lower}}_query. with input parameters hash_key, range_key_and_condition, filter_key_condition, consistent_read, index_name, query_limit, scan_index_foward, last_evaluated_key, attribute_to_get. hash_key is mandatory parameter. scan_index_forward is True by default and consistent_read is False by default. query the table with these input parameters. Method should query the table using inputs to the function till last_evaluated_key is None. Method should output the list of {{ddb_table_name}}Dto.
12) Seventh method is wrapper around sixth method taking input table’s partition/hash key (pk), range_key_and_condition and filter_key_condition. Method should output the list of {{ddb_table_name}}Dto and last_evaluated_key in result. Name of the method is {{ddb_table_name_lower}}_query_base_table
13) Eight method is to batch write into base table. Name of the method is {{ddb_table_name_lower}}_batch_write. if there is batch write failure then process the failed item in the batch one by one using the methods created earlier and identify the erroring items and store in a error_items list and raise the error at the end if error_items list has items{{created_at_replacement}}{{updated_at_replacement}}. {{time_to_live_method_replacement}}
14) Ninth method is to batch get from base table. Name of the method is {{ddb_table_name_lower}}_batch_get
15) If GSI is present, Create methods(one per GSI) is warpper around sixth method taking input gsi_name as index_name, gsi’s partition/hash key (pk), range_key_and_condition and filter_key_condition. Method should output the list of {{ddb_table_name}}Dto and last_evaluated_key in result. Name of the method is gsi name concatenation with "_query"
16) If LSI is present, Create methods(one per LSI) is wrapper around sixth method taking input lsi_name as index_name, table’s partition/hash key (pk), range_key_and_condition and filter_key_condition. Method should output the list of {{ddb_table_name}}Dto and last_evaluated_key in result. Name of the method is lsi name concatenation with "_query"
